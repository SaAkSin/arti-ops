# 📄 [PRD] arti-ops v0.5.2 제품 요구사항 명세서 (전면 개정판)

## 1. 제품 개요 (Product Overview)

* **제품명:** arti-ops (로컬 컨텍스트 기반 AgentOps TUI 플랫폼)
* **목적:** 사용자가 현재 작업 중인 경로(디렉토리)에서 명령어를 실행하면, **사내 위키(BookStack)의 범용 정책과 현재 로컬에 작성된 파일들(현황, 아키텍처, 언어)을 교차 분석**하여, 사용자의 프롬프트 지시에 따라 대상 워크스페이스에 딱 맞게 결합(Merge)된 스킬(Skill)과 룰(Rule)을 자동 생성 및 검증하는 전체화면 대화형 TUI 환경.
* **핵심 가치:** 단순한 정책 템플릿 다운로드 도구를 넘어, 로컬 코드 문맥을 이해하고 위키의 진실의 원천(Source of Truth)을 융합하여 개발자 맞춤형 AI 스킬/룰을 지능적으로 창작(Workspace Context Fusion)하고 안전하게 배포하는 어시스턴트.

## 2. 핵심 기능 요구사항 (Functional Requirements)

* **F1. 위키-로컬 융합 프롬프트 주도 생성 (Workspace Context Fusion):**
  * 시스템은 사내 위키(BookStack)의 Global/Workspace 정책과 **명령어가 실행된 현재 경로의 로컬 파일 구조 및 기존 산출물들을 스캔하여 컨텍스트로 참조**한다.
  * 이 두 가지 지식을 융합하여, 위키의 내용을 덮어쓰거나 무시하지 않고 현재 워크스페이스 성격에 맞게 변형/병합(Merge)하여 타겟 에이전트(`antigravity`)가 이해할 수 있는 규격(`.agents/skills/`, `.agents/rules/`)으로 생성한다.
  * 사용자는 하단 프롬프트 입력창을 통해 자연어로 지시를 내리며, 특히 `s 1` (또는 `시작`) 단축 매크로를 입력 시 AI가 알아서 프로젝트 상태를 분석하고 가장 필요한 룰/스킬을 스스로 기획한다.
* **F2. 자율 정책 검증 (Policy Verification):**
  * 생성된 스킬과 룰이 회사의 L1(전역) 보안 정책이나 가이드라인을 위반하지 않는지 철저히 검증한다.
* **F3. 사전 검토 보고서 제출 및 반영 판단 (Pre-deployment Review):**
  * 검증이 완료되더라도 즉시 로컬 파일로 저장하지 않는다.
  * 생성된 파일들의 **정확한 목록과 전체 코드 내용, 검증 결과가 상세히 정리된 [최종 보고서]**를 화면 중앙에 출력한다.
  * 사용자는 이 보고서를 읽고 하단 프롬프트를 통해 승인(반영) 여부를 판단하거나, 추가 수정 지시(재생성 요구)를 내릴 수 있다.
* **F4. 전체화면 대화형 TUI 및 Native IME (Full-screen Console):**
  * 콘솔의 전체화면(Full-screen)으로 동작하며, 한글 입력을 완벽히 지원하기 위해 기본 터미널(Native IME) 프롬프트 시스템을 사용한다.
  * 화면 상단/중앙은 실시간 파이프라인 진행 상태 트리가, 하단은 사용자의 프롬프트와 메세지가 출력된다.
* **F5. 즉시 종료 스위치 (Kill-switch):**
  * `q` (quit) 문자 입력 또는 `Ctrl+C` 키 입력 시, 파이프라인이 어떤 작업을 하고 있든 관계없이 무조건 즉시 안전하게 종료(Exit)되어야 한다.
* **F6. GWS 채팅 알림 간소화:**
  * 기존의 중간 결재/대기용 GWS Chat 연동 로직은 폐기한다.
  * 오직 로컬 배포가 사용자의 승인 하에 성공적으로 완료된 직후, **최종 요약 보고서**만을 GWS Chat으로 단방향 알림 전송한다.
* **F7. 세션 캐시 관리 및 시각화 (Session Cache Management):**
  * LLM이 이전 대화 컨텍스트(`sessions.db`)를 재활용하여 API 호출을 생략(Cache Hit)하는 경우, 명시적으로 `💡 이전 세션(sessions.db) 기억을 불러왔습니다.` 형태의 메시지를 트리에 출력하여 사용자에게 인지시킨다.
  * 프롬프트에 `r` 또는 `reset`을 입력하면 즉시 세션 DB를 파기하고 파이프라인을 완전 백지 상태로 초기화할 수 있다.
* **F8. 로컬-위키 연동 대화형 배포 (BookStack Upsert):**
  * 프롬프트에 `u` 또는 `upsert` 입력 시 로컬 `.agents` 디렉토리의 파일들과 대상 워크스페이스의 위키 페이지 현황을 스캔 및 비교한다.
  * 배포 예정인 파일 목록(생성/업데이트 여부)을 체크박스 형태의 다이얼로그로 화면에 띄워, 사용자가 방향키와 스페이스바로 원하는 항목만 취사선택하여 안전하게 BookStack에 반영할 수 있다.
* **F9. 로컬 에셋 현황 직관적 조회·편집·L1 변환 뷰어 (`l` 명령어):**
  * 프롬프트에 `l` 또는 `list` 입력 시, 즉시 전체화면(Full-screen) TUI 뷰어 모드로 전환된다.
  * **좌우 분할(Split) 레이아웃:** 좌측 패널에는 `.agents` 하위의 `rules`와 `skills` 정보 목록이 계층 구조로 표시되며, 위키 현황 비교를 통해 신규(`!`) / 변경(`*`) / 일치(` `) 배지가 시각적으로 표시된다.
  * **대화형 조작:** `↑`/`↓` 이동 시 자동으로 우측 패널에 파일 미리보기 로드. `Enter`로 즉시 편집 모드 진입(`Ctrl+S` 저장, `Esc` 취소). `Tab`으로 패널 포커스 전환.
  * **`g` 키 (L1 변환 미리보기):** 현재 파일을 ADK `l1_globalizer` 에이전트(Gemini PRO)로 L1 전역 정책으로 변환하여 우측 패널에 표시. 스킬의 Scripts 섹션은 예제 형식으로 변환되며 L3 머지 안내 주석이 자동 삽입된다. `Esc`로 원본 복원. `Ctrl+C`/`⌘+C`로 클립보드 복사.
  * **`u` 키 (위키 배포):** BookStack 연결 시 현재 워크스페이스의 신규·변경 항목을 체크박스 다이얼로그로 선택하여 안전하게 배포 가능. 배포 완료 후 배지 자동 갱신.
  * `q`/`Esc`를 누르면 기존 메인 뷰어 상태로 복귀한다.
