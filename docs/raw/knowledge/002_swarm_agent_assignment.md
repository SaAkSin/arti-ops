# 002. 스웜 팩토리 티켓 팩토리: 담당 에이전트 명시 원칙

## 배경 상황 (Incident Context)
- 작업 일자: 2026-04-20
- 쟁점: `arti-pm`, `arti-auto` 등 PM 및 Auto-pilot 역할 수행 중 백로그 티켓 생성 시 담당 에이전트 메타데이터를 누락하는 습관 발견.
- 원인: 로컬 단일 환경에서 빠르게 코드를 작성하고 푸시하기 위해 AI 에이전트 자신이 모든 작업을 도맡아 하려는 (1인 다역) 편의성을 선택함. 이로 인해 물리적 워크트리 분배에 필수적인 라우팅 태그(`@arti_ui`, `@arti_core` 등)가 무시됨.

## 학습 결과 (Lesson Learned)
마이크로 티켓 팩토리는 단순한 TODO 리스트가 아니라, **분산된 스웜 에이전트들이 자신의 워크트리 폴더로 일감을 퍼가기 위해 사용하는 통신 큐(Message Queue) 명세서**입니다. 담당자가 없는 티켓은 시스템의 격리성 규칙 아키텍처를 근본적으로 파괴합니다.

### 행동 지침 (Action Items)
1. **담당 에이전트 필수 기재**: 새로운 마크다운 티켓을 1-backlog에 발행할 때 규칙에 어긋나지 않도록 항상 티켓 내용 상단 또는 제목에 예: `AssignedTo: @arti_cli` 혹은 `담당: @arti_ui` 처럼 에이전트를 명확히 기재하십시오.
2. **단독 구동 편파 방지**: '나 혼자서 main 브랜치에서 다 처리할 수 있다'는 식의 가정을 시스템에서 지워버리십시오. 티켓은 반드시 시스템 가이드에 명시된 에이전트 종류 중 가장 적합한 역할을 지닌 에이전트로 배정되어야 합니다.
