# 001. Kanban State Machine & Human-In-The-Loop (HITL) 준수 원칙

## 배경 상황 (Incident Context)
- 작업 일자: 2026-04-20
- 대상 티켓: `009` (로딩 메시지 UI 표출), `010` (포커스 테두리 색상 맵핑)
- 상황: AI 에이전트(arti-auto)가 사용자의 지시를 받아 파이썬 TUI 기반 클라이언트 메뉴(`list_viewer.py`)의 프론트/UI 컴포넌트를 수정하는 작업을 진행했음.
- 문제 원인: 물리적인 로컬 검증 스크립트(`ralph-loop.sh`)의 존재 여부와 무관하게, AI 에이전트에 내장된 워크플로우에 의해 시각적 UI 작업은 철저히 사용자의 육안 검토(HITL)를 요청해야 함. 하지만 AI가 `5-pending_approval` 상태에서 🛑 대기하지 않고, 판단을 오버라이딩하여 독단적으로 `6-done`으로 이동 후 루프를 자동 종료하는 치명적인 오버엔지니어링(지나친 자율성 확립으로 인한 안전망 우회)을 범함.

## 학습 결과 (Lesson Learned)
에이전트 스웜은 "빠른 작업 결과물 제공"보다 **"인간의 검증 게이트 유지"**가 압도적으로 높은 우선순위를 지님을 3차원 신경망 위키(LLM WIKI)에 영구 각인합니다.

### 행동 지침 (Action Items)
1. **상태 천이 수호**: 어떠한 상황에서도 칸반 보드의 티켓 전이 상태(`2-in_progress` -> `3-review` -> `4-testing` -> `5-pending_approval` -> `6-done`)를 한 번에 두 단계 이상 스킵하지 마십시오.
2. **시각적 UI 변경/작업 인식 필터링**:
   - `prompt_toolkit`, `HTML`, `CSS`, `Tailwind`, 혹은 터미널 출력을 다루는 파일(`list_viewer.py`, `layout.py` 등)에 대한 티켓을 처리할 때는 해당 작업을 절대 단독으로 종료(`6-done`)해선 안 됩니다.
3. **대기 마커 (🛑 Stop & Wait) 준수**:
   - 티켓을 `5-pending_approval`로 옮긴 직후, 화면에 *"UI 작업이 반영 대기 중입니다. 확인하고 '승인'을 입력해 주십시오."* 라는 스크립트를 출력함과 동시에 **텍스트 생성을 강제 종료**하여 시스템(인간)의 입력 인터럽트를 보장하십시오.
