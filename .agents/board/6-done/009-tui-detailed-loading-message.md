# 009-tui-detailed-loading-message
DependsOn: [None]

## 작업 목적
`l` 명령어 최초 1회 실행(캐시 미스 상태) 시 표출되는 로딩 메시지가 단순히 "BookStack 로컬 현황 및 정책을 로드 중입니다..."로 고정되어 있어 사용자가 대기(지루함)를 체감할 수 있습니다. 이를 해결하기 위해 네트워크 페칭 구간을 세부 단계(Phase)로 나누어 구체적인 진행 상황을 알 수 있도록 UI를 개선합니다.

## 변경 요건
1. `src/arti_ops/cli/main.py` 파일 내부의 캐시가 없을 때 동작하는 로딩 로직을 수정합니다.
2. 기존에 `asyncio.gather`를 통해 한 번의 `console.status` 안에서 병렬 처리되던 방식을 유지하되, Rich `Progress` 바 시스템을 사용하거나, 각 페칭 작업을 순차(Sequential)로 나누어 `console.status.update()`를 통해 텍스트가 "Step 1: 로컬 에셋(Upsert Plan)을 분석하고 있습니다...", "Step 2: 글로벌(G1) 지식 그래프를 맵핑 중입니다..." 식으로 동적으로 갱신되도록 변경하십시오.

## 검증 방법
- `r` (캐시 초기화) 명령어를 통해 캐시를 비웁니다.
- 프롬프트에 `l`을 입력합니다.
- TUI 화면이 렌더링되기 전 콘솔에 나타나는 로딩 인디케이터가 1단계, 2단계 등 세부 작업의 형태나 순서에 맞게 구체적인 문장으로 변하며 진행되는지 확인합니다.
