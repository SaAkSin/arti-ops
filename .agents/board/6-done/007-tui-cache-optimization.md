# 007-tui-cache-optimization
DependsOn: [None]

## 작업 목적
사용자가 TUI에서 `l` (List) 명령어로 진입할 때마다 `_viewer_cache.clear()`가 동작하여 로컬 DB 캐시가 매번 삭제되고 무거운 위키 API 폴링이 스레드를 막는 병목(Latency) 현상을 해결합니다.

## 변경 요건
1. `src/arti_ops/cli/main.py` 파일의 `l` 명령어 분기 내에서 동작하는 `_viewer_cache.clear()` 구문을 제거하거나 주석 처리하십시오.
2. 대신, 사용자가 [r] 혹은 [reset] 명령어를 입력했을 때에만 캐시가 완전하게 지워지도록 동작을 확인 및 분리하십시오. (이미 r 명령어에 세션 DB 삭제 로직이 존재하므로 정책 캐시도 동일하게 초기화되도록 동기화)

## 검증 방법
- `arti-ops` 앱 내에서 프롬프트에 `l`을 두 번 연달아 입력합니다.
- 첫 로딩 이후, 두 번째 `l` 진입부터는 위키 페이지 로딩 과정이 캐시를 타서 1초 이내에 즉각적으로 화면이 열려야 합니다.
