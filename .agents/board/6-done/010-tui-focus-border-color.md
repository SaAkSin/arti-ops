# 010-tui-focus-border-color
DependsOn: [None]

## 작업 목적
TUI 뷰어(`l` 명령어 실행 상태)에서 사용자가 포커스 이동 트리거(예: `Tab` 키)를 통해 패널 간 이동 시, 어느 패널이 포커스 상태인지 보다 현대적으로 인지할 수 있도록 타이틀 옆의 `(포커스됨)` 텍스트를 제거하고, 대신 포커스된 뷰의 테두리(Border) 색상이 눈에 띄게 변경되도록 UI 프레젠테이션 로직을 개선합니다.

## 변경 요건
1. `src/arti_ops/cli/list_viewer.py`에서 패널 포커스 변경 시 타이틀 텍스트에 "(포커스됨)" 문자열을 붙였다 떼었다 하는 하드코딩 로직(예: `update_titles()` 등)을 찾아 기존 패널명을 순수 텍스트로 고정 유지하도록 수정하십시오.
2. `prompt_toolkit`의 `Frame` 컴포넌트 `style` 속성에 동적 상태(함수 콜백 `lambda: ...` 방식 등)를 주입하거나 렌더링 타이밍마다 스타일이 업데이트되도록 하여, 현재 앱 레이아웃에서 `get_app().layout.has_focus()` 결과에 따라 포커스된 뷰(예: 목록창 혹은 본문창)의 테두리와 타이틀 색상이 돋보이는 색깔(예: `class:frame.focused` 로 지정하고 `cyan`이나 `yellow` 적용)로 동적 렌더링되게 구현하십시오.

## 검증 방법
- `l` 뷰어를 실행합니다.
- 좌측 마이크로스킬 목록에 포커스가 있을 때, 좌측 보더 라인이 강조색으로 표시되고 타이틀에 "포커스됨" 문구가 뜨지 않아야 합니다.
- `Tab` 키(또는 `Enter` 키로 편편집 이동)를 통해 우측 패널로 포커스가 넘어가면, 좌측 보더가 기본색으로 돌아가고 우측 보더가 강조색으로 바뀌어야 합니다.
