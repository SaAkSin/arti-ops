# 011-tui-grey-out-uneditable
DependsOn: [None]

## 작업 목적
TUI 뷰어(`l` 명령어)의 좌측 패널(로컬 파일 목록)에서 사용자가 편집(Enter/e)할 수 없는 항목들을 시각적으로 명확히 구분하기 위해, 편집 불가능한 항목의 글자 색상을 회색(Grey) 계열로 디밍(Dimming) 처리합니다.

## 변경 요건
1. `src/arti_ops/cli/list_viewer.py` 내의 좌측 패널 아이템 렌더링 로직(`update_left_pane` 및 `update_toolbar` 등)을 분석하십시오.
2. 현재 로직 상 편집이 불가능한 항목 (예를 들어, `remote://` 형태의 원격 정책 매핑 파일이거나 `.md`가 아닌 파일, `Match` 판정을 받아 배포본과 동일한 항목 등 프로젝트 정책에 따라 편집 창(에디터) 진입이 차단된 항목)을 식별하는 조건을 도출하십시오.
3. `prompt_toolkit`의 `FormattedText`나 `HTML` 태그 등을 활용하여, 위 조건에 해당하는(편집 불가한) 항목은 목록에서 표시될 때 텍스트 색상이 어두운 회색(`<style fg="darkgray">` 등)으로 시각화되도록 UI를 수정하십시오.
4. 방향키 이동으로 선택(`selected`)된 상태일 때의 `bg:#00aa00 #ffffff bold` 하이라이팅 스타일과는 충돌하지 않도록 스타일의 우선순위를 꼼꼼히 확인하십시오.

## 자율 검증 기준 (Ralph Loop)
- TUI 뷰어를 띄웠을 때 로컬(`.md`) 등 편집 가능한 파일은 흰색/지정된 원문 색상으로 보이고, 편집이 불가능한 항목(G1 정책 매핑 템플릿 등)은 회색으로 디밍되어 보이는지 확인합니다.
- 회색으로 표시된 항목에 포커스를 올렸을 때 백그라운드 초록색 선택 박스 렌더링이 깨지지 않아야 합니다.
