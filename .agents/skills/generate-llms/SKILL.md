# Role
당신은 프로젝트의 전체 코드베이스를 AI 친화적인 단일 텍스트 컨텍스트 문서(`llms.txt`)로 병합하는 문서 관리 에이전트입니다. 

# Rules & Constraints
- **보안 및 정합성**: 환경 설정 파일(`.env`), AI 관련 프롬프트/스크립트(`.agents`), 락 파일(`.lock`), 정적 에셋은 철저히 제외되어야 합니다. 이 정규식 필터링 로직은 종속된 스크립트 내부에 내장되어 있습니다.
- **최신화 타이밍**: 시스템에 새로운 핵심 기능(도메인)이 추가되거나 대규모 리팩토링 구조 변경이 있을 때, LLM이 전체 구조를 정확히 인지할 수 있도록 컨텍스트를 즉시 새롭게 병합(갱신)해야 합니다.

# Action Guidelines
1. 사용자가 별도로 프로젝트 컨텍스트(LLM 파인튜닝/RAG용 문서) 갱신을 요청하거나, 개발 작업이 끝난 후 본인이 새 컨텍스트 빌드가 필요하다고 판단할 때 이 스킬이 개입합니다.
2. 실행 시 `.agents/skills/generate-llms/scripts/generate.sh` 쉘 스크립트를 Bash 환경에서 직접 실행하십시오. 스크립트가 알아서 프로젝트 루트에서 안전하게 동작합니다.
3. 실행이 정상 종료되면 작업 공간의 `docs/llms.txt` 경로에 산출물이 덮어씌워져 저장됩니다. 
4. 작업 완료 후 총 몇 개의 소스 파일이 병합되었는지 스크립트 출력(Output)을 읽어 사용자에게 심플하게 보고하십시오.
